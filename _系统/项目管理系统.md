# PDCA项目管理系统

## 系统入口检测逻辑

每次进入PDCA项目系统时，AI必须执行以下检测：

### 1. 目录检测
```bash
pwd
ls -la 项目/
```

### 2. 项目状态读取
读取项目状态跟踪文件，了解：
- 新建项目数量
- 进行中项目列表
- 已完成项目列表

### 3. 用户指令提示
在系统启动时，必须显示以下三个指令：

```
🚀 PDCA项目管理系统

可用命令：
- new     ：启动新项目
- ongoing : 选择进行中的项目
- achieve : 选择已完成的项目

请输入命令：
```

## 项目状态定义

### 新项目（New）
- 创建时间 < 1天
- 处于Plan阶段初期
- 尚未进入实质执行阶段

### 进行中项目（Ongoing）
- 处于Plan/Do/Check/Act任一阶段
- 有具体的工作进展
- 未完成全部四个阶段

### 已完成项目（Achieve）
- 完成Act阶段
- 完成项目总结
- 进入经验库

## 项目切换流程

### new命令流程
1. 提示用户输入项目名称
2. 创建项目文件夹结构
3. 生成项目信息文件
4. 启动PDCA控制器
5. 进入Plan阶段

### ongoing命令流程
1. 显示所有进行中的项目列表
2. 用户选择项目
3. 读取项目当前状态
4. 显示项目进展
5. 根据当前阶段加载对应Agent

### achieve命令流程
1. 显示所有已完成的项目列表
2. 用户选择项目
3. 显示项目总结
4. 提供经验借鉴功能
5. 支持创建类似项目

## 项目状态文件

项目状态必须通过项目信息.md文件中的"项目状态"字段管理：

- 新建项目：项目状态 = "新建"
- 进行中项目：项目状态 = "进行中"
- 已完成项目：项目状态 = "已完成"

## 系统响应模板

### 系统启动
```
🚀 PDCA项目管理系统

可用命令：
- new     ：启动新项目
- ongoing : 选择进行中的项目
- achieve : 选择已完成的项目

当前有：
- 新建项目：X个
- 进行中项目：Y个
- 已完成项目：Z个

请输入命令：
```

### new响应
```
✨ 启动新PDCA项目

请输入项目名称：
```

### ongoing响应
```
📋 进行中的项目列表：

1. [项目名称1] - Plan阶段（第X天）
2. [项目名称2] - Do阶段（第Y天）
3. [项目名称3] - Check阶段（第Z天）

请选择项目编号（1-3）：
```

### achieve响应
```
🏆 已完成项目列表：

1. [项目名称1] - 完成日期：2025-XX-XX
2. [项目名称2] - 完成日期：2025-XX-XX

请选择项目编号（1-2）：
```

## 项目上下文切换

选择项目后，系统必须：
1. 切换到项目文件夹
2. 读取项目信息.md
3. 识别当前阶段
4. 加载对应的Agent
5. 显示项目当前状态
6. 等待用户输入下一步指令

## 注意事项

1. **一致性**：所有响应必须使用相同的格式和术语
2. **清晰性**：项目状态描述必须清晰准确
3. **连续性**：切换项目后保持上下文连续
4. **引导性**：主动提示用户可执行的操作
