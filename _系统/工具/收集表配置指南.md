# Bitable 收集表配置指南

## 概述

Bitable 收集表用于用户快速录入数据，提交后数据自动写入数据记录表。
每个 MECE 框架的维度配置一个独立的收集表。

## 创建流程

```
项目创建时 (PlanAgent Phase 4)
    ↓
根据选择的 MECE 框架 → 获取维度列表
    ↓
为每个维度创建对应的收集表
    ↓
收集表字段 → 绑定到数据记录表的 指标名称/维度/数值/单位
    ↓
分享收集表链接 → 用户通过链接填写
```

## 各框架收集表配置

### TREND 框架（健康/体能）

| 维度 | 收集表名称 | 字段配置 |
|------|-----------|---------|
| Training（训练） | 训练数据收集表 | 训练时长(分钟,数字)、训练强度(单选:低/中/高)、训练类型(多选) |
| Rest（休息） | 休息数据收集表 | 睡眠时长(小时,数字)、睡眠质量(单选:1-5分)、休息日标记(是/否) |
| Eating（营养） | 营养数据收集表 | 热量摄入(kcal,数字)、蛋白质(g,数字)、碳水化合物(g,数字) |
| Nature（身体） | 身体数据收集表 | 体重(kg,数字)、静息心率(bpm,数字)、VO2max(数字) |
| Daily（日常） | 日常活动收集表 | 步数(数字)、久坐时长(分钟,数字) |

### 4M1E 框架（制造业）

| 维度 | 收集表名称 | 字段配置 |
|------|-----------|---------|
| Man（人员） | 人员数据收集表 | 培训时长(小时)、技能等级(单选)、出勤(数字) |
| Machine（设备） | 设备数据收集表 | 运行时长(小时)、故障次数(数字)、维护状态(单选) |
| Material（材料） | 材料数据收集表 | 来料数量(数字)、不合格数(数字)、供应商(单选) |
| Method（方法） | 工艺数据收集表 | 工艺参数(数字)、合格率(百分比)、SOP版本(单选) |
| Environment（环境） | 环境数据收集表 | 温度(°C)、湿度(百分比)、5S评分(数字) |

### 其他框架

其他框架（PPTD, 5P, COMET, 3RL-TD, GRCT, 5S, TIME, SIPOC, 5P2E）的收集表配置在项目创建时根据维度动态生成。

## 数据写入规则

1. 收集表提交时，数据自动写入数据记录表
2. 字段映射：
   - 收集表"指标"字段 → 数据记录表"指标名称"字段
   - 收集表"维度"固定值 → 数据记录表"维度"字段
   - 收集表"数值"字段 → 数据记录表"数值"字段
   - 收集表"单位"固定值 → 数据记录表"单位"字段
3. 系统自动填充字段：
   - 记录时间：当前时间戳
   - 记录人：提交用户

## API 创建方式

通过 Bitable 工作流编辑器或飞书 API 创建收集表。

**注意**: Bitable 收集表的 API 创建方式需要验证。如果 API 不支持，需要在 Bitable UI 中手动配置，并在文档中记录配置步骤。

## API 调用注意事项

- 收集表创建后需要获取分享链接发送给用户
- 收集表字段类型应与数据记录表对应字段类型一致
- 每个维度的收集表应独立配置，避免字段混淆
