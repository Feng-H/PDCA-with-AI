# 阶段转换检查表

确保PDCA循环在正确的时间点进行阶段转换，避免过早推进或延误。

---

## 📋 Plan → Do 转换检查表

### 核心成果检查（必须全部完成）
- [ ] **问题定义文档**已确认
  - 问题描述清晰准确
  - 根本原因已识别
  - 影响范围已评估

- [ ] **目标设定文档**已确认
  - 符合SMART原则
  - 量化指标已定义
  - 里程碑已设定

- [ ] **解决方案文档**已确认
  - 方案可行性已验证
  - 备选方案已准备
  - 实施路径已明确

- [ ] **执行计划文档**已确认
  - 任务分解完整
  - 时间节点合理
  - 责任人已指定

- [ ] **资源保障**已确认
  - 人力资源到位
  - 物质资源准备
  - 预算已批准

### 时间检查
- [ ] Plan阶段时间使用合理（不超过计划的110%）
- [ ] Do阶段开始时间已确定
- [ ] 整体项目时间线仍可行

### 风险检查
- [ ] 主要风险已识别并有应对措施
- [ ] 关键假设已验证
- [ ] 外部依赖已确认

### 转换决策
```
如果所有条件满足 → 立即转换到Do阶段
如果部分条件满足 → 延长Plan阶段，补充缺失部分
如果关键条件缺失 → 重新评估项目可行性
```

---

## 📋 Do → Check 转换检查表

### 执行完成度检查
- [ ] **计划任务100%完成**
  - 所有任务标记为完成
  - 交付物已验收
  - 质量标准已达标

- [ ] **过程记录完整**
  - 执行日志已记录
  - 问题处理过程已文档化
  - 偏差调整已说明

- [ ] **数据收集完成**
  - 关键指标数据已收集
  - 过程数据已整理
  - 异常数据已标注

### 时间检查
- [ ] Do阶段在计划时间内完成（不超过120%）
- [ ] 时间偏差已记录并分析原因
- [ ] 后续阶段时间影响已评估

### 质量检查
- [ ] 交付质量符合标准
- [ ] 客户/用户满意度达标
- [ ] 返工率在可接受范围

### 转换决策
```
如果执行完成且数据完整 → 转换到Check阶段
如果执行未完成 → 延长Do阶段
如果计划问题导致失败 → 考虑回退到Plan阶段
```

---

## 📋 Check → Act 转换检查表

### 分析完成度检查
- [ ] **数据分析完成**
  - 目标达成度已计算
  - 差异分析已完成
  - 趋势分析已进行

- [ ] **效果评估完成**
  - 定量评估已完成
  - 定性评估已完成
  - 成本效益分析已完成

- [ ] **问题诊断完成**
  - 主要问题已识别
  - 根本原因已分析
  - 影响因素已确定

- [ ] **改进建议已提出**
  - 具体改进措施已制定
  - 优先级已排序
  - 实施建议已明确

### 时间检查
- [ ] Check阶段时间使用合理
- [ ] 分析深度与时间投入匹配
- [ ] Act阶段时间需求已评估

### 结论可靠性检查
- [ ] 数据支撑充分
- [ ] 分析方法正确
- [ ] 结论逻辑清晰
- [ ] 建议切实可行

### 转换决策
```
如果分析完整且结论可靠 → 转换到Act阶段
如果数据不足 → 考虑补充执行（回退到Do）
如果目标或方案问题 → 考虑重新规划（回退到Plan）
```

---

## 📋 Act → 结束/新循环 转换检查表

### 行动完成度检查
- [ ] **决策已制定**
  - 基于分析的明确决策
  - 决策理由充分
  - 决策风险可控

- [ ] **行动方案已实施**
  - 具体措施已执行
  - 实施效果已确认
  - 问题已解决或改善

- [ ] **知识沉淀完成**
  - 经验教训已总结
  - 最佳实践已提炼
  - 文档已更新

### 项目目标达成检查
- [ ] **原问题已解决**
  - 问题症状消除
  - 根本原因已处理
  - 效果可持续

- [ ] **目标已达成**
  - 量化目标已实现
  - 质量标准已满足
  - 相关方满意度达标

### 循环必要性评估
- [ ] **未解决问题识别**
  - 新问题已识别
  - 问题优先级评估
  - 解决紧迫性分析

- [ ] **改进机会识别**
  - 进一步优化空间
  - 成本效益评估
  - 资源可用性

### 最终决策
```
如果问题完全解决且无进一步改进需求 → 项目结束
如果部分解决或有改进空间 → 启动新PDCA循环
如果外部因素影响 → 暂停项目
```

## 🔄 回退决策检查表

### 何时考虑回退

#### Do → Plan 回退
检查是否出现：
- [ ] 计划执行不可行
- [ ] 外部环境重大变化
- [ ] 资源不可获得
- [ ] 目标需要重新定义

#### Check → Do 回退
检查是否出现：
- [ ] 数据不充分无法分析
- [ ] 执行过程有重大遗漏
- [ ] 需要补充实验或测试
- [ ] 原始数据质量问题

#### Check → Plan 回退
检查是否出现：
- [ ] 目标设定不合理
- [ ] 解决方案根本缺陷
- [ ] 初始问题理解错误
- [ ] 约束条件重大变化

#### Act → 前序阶段回退
检查是否出现：
- [ ] 行动实施遇到根本障碍
- [ ] 决策基础存在问题
- [ ] 需要重新分析或规划

### 回退成本评估
- [ ] 时间成本可接受
- [ ] 资源成本可控
- [ ] 机会成本合理
- [ ] 回退收益明确

### 回退执行检查
- [ ] 当前成果已保存
- [ ] 回退路径明确
- [ ] 回退后重新规划
- [ ] 避免重复工作

## ⏰ 时间管理提醒

### 阶段时间预警
- **80%时间使用**：黄色预警，关注进度
- **100%时间使用**：橙色预警，加速或调整
- **120%时间使用**：红色预警，必须决策

### 时间偏差处理
- **轻微偏差（<10%）**：记录原因，继续推进
- **中度偏差（10-30%）**：调整计划，加强管控
- **严重偏差（>30%）**：评估是否需要回退

### 阶段重叠考虑
- 某些情况下允许阶段轻微重叠
- 重叠时间不超过20%
- 明确重叠期的工作重点
