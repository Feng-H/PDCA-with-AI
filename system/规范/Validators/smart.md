# SMART 目标校验标准

本模块用于 AI Agent 自动检查 PDCA 项目中的目标设定是否符合 SMART 原则。

---

## 📋 校验维度

### 1. Specific (明确性)
- **定义**：目标必须是具体的，不能笼统。
- **检查项**：
    - [ ] 是否明确了“做什么”？
    - [ ] 是否明确了具体的对象或范围？
    - [ ] 描述是否具有唯一解释性，无歧义？
- **错误示例**：
    - “提高生产效率” (太笼统)
    - “优化系统性能” (未指定具体模块)

### 2. Measurable (可衡量性)
- **定义**：目标必须是可量化的，有明确的评价指标。
- **检查项**：
    - [ ] 是否包含具体的数值或百分比？
    - [ ] 指标的数据来源是否明确（如：来自项目的 OEE 数据表或采集系统）？
    - [ ] 是否定义了计算公式？
- **错误示例**：
    - “显著提升质量” (无量化指标)
    - “让客户满意” (缺乏衡量手段)

### 3. Achievable (可达成性)
- **定义**：目标在当前资源和能力范围内是可实现的。
- **检查项**：
    - [ ] 目标增幅是否符合行业或历史规律（如：通常不建议单次循环提升 >50%）？
    - [ ] 是否识别了必要的资源支撑？
    - [ ] 逻辑上是否存在不可逾越的障碍？
- **错误示例**：
    - “零故障运行” (理想化，通常不可持续达成)

### 4. Relevant (相关性)
- **定义**：目标必须与上层痛点或核心问题直接相关。
- **检查项**：
    - [ ] 目标达成后，是否能直接缓解“项目信息”中描述的痛点？
    - [ ] 是否与当前 PDCA 循环的主题保持一致？
- **错误示例**：
    - 痛点是“设备开裂”，目标设定为“降低备件库存”。

### 5. Time-bound (时限性)
- **定义**：目标必须有明确的截止日期或时间周期。
- **检查项**：
    - [ ] 是否明确了“何时完成”？
    - [ ] 时间节点是否与 PDCA 的 `Plan` 节奏匹配？
- **错误示例**：
    - “尽快完成改善” (无具体日期)

---

## 🤖 AI 判定指令集

当 AI Agent 进行校验时，应输出如下格式：

> **SMART 校验结果：**
> - **状态**：✅ 通过 / ❌ 驳回
> - **缺失项**：[列出不符合的维度]
> - **修改建议**：[给出具体的修改意见，例如：建议将“提高效率”改为“在 2024-06-30 前，将 A 线 OEE 指标从 65% 提升至 75%”]
