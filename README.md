# PDCA项目管理系统

**基于PDCA循环的AI增强问题解决系统** — 专注于制造和质量管理领域的持续改进

受到 https://github.com/Tasihi89/vibe-writing-workflow 项目的启发而开发，感谢 Tasihi89

## 🎯 系统简介

PDCA项目管理系统不是简单的AI问答工具，而是一个基于**时间流驱动**的**问题解决工作流系统**。它将经典的PDCA循环与AI能力结合，帮助用户高效解决制造和质量问题。

### 核心特点
- **严格时间序列**：Plan→Do→Check→Act必须按时间顺序推进
- **阶段转换机制**：明确的转换条件和决策点
- **多项目管理**：同时管理多个PDCA项目
- **知识沉淀**：将成功经验标准化，便于复用
- **🆕 主动驱动**：通过 OpenClaw 定时任务和心跳机制主动推进项目

## 🆕 OpenClaw AgentSkill 版本

### 为什么要做成 Skill？

原始的 PDCA 系统只是一组 Markdown 文件和提示词模板，存在一个根本问题：**它无法主动触发**。

传统的 AI 助手模式是"用户问，AI 答"——如果你今天不打开对话框，PDCA 流程就完全停滞。这对项目管理来说是致命的：阶段到期了没人提醒、里程碑到了没人跟进、项目超时了也没人知道。

**OpenClaw 提供了解决方案：**

OpenClaw 是一个 AI Agent 运行时平台，支持两种主动驱动机制：
- **Cron 定时任务**：在指定时间自动触发 AI 执行任务
- **Heartbeat 心跳**：定期轮询，让 AI 检查是否有需要处理的事项

通过将 PDCA 系统改造为 OpenClaw 的 AgentSkill，系统能够：
1. **每天早上9点自动提醒**当前阶段待办和剩余时间
2. **阶段到期前自动预警**，提示检查转换条件
3. **里程碑当天自动提醒**待交付内容
4. **AI 主动读取飞书文档**，判断模板填写进度，推断项目状态
5. **关键节点推动用户决策**，而不是被动等待

这就是从"文件模板"到"主动驱动工作流"的本质升级。

### 运行机制

```
用户描述问题
    ↓
AI 评估是否适合 PDCA（先聊背景，不硬推）
    ↓ 推荐
用户确认
    ↓
自动创建飞书知识库项目结构（Plan/Do/Check/Act 文件夹 + 模板文档）
    ↓
注册 Cron 定时任务（每日提醒 + 阶段预警 + 里程碑提醒）
    ↓
用户在飞书填写文档
    ↓
AI 主动读取文档 → 判断完成度 → 提醒/推动阶段转换
    ↓
用户确认转换 → AI 更新 Cron → 进入下一阶段
    ↓
循环直到项目完成 → 清理 Cron → 归档经验
```

### 主动驱动详情

| 驱动方式 | 频率 | 行为 |
|---------|------|------|
| Cron 每日提醒 | 每天 09:00 | 读取项目状态，提醒当前阶段待办、剩余时间、到期预警 |
| Cron 阶段预警 | 阶段结束前1天 | 提示检查阶段转换条件 |
| Cron 里程碑提醒 | 里程碑当天 | 提醒待交付内容 |
| Heartbeat | 每次心跳轮询 | 轻量检查：有无活跃项目、有无到期事项、有无超时项目 |

**决策权始终在用户手中。** AI 只负责推动和信息，阶段转换、项目结束等关键决策必须用户确认。

### 判断项目进展（方案C）

AI 通过两种方式结合判断项目进展：

1. **主动读取飞书文档**：检查模板中的必填项是否已填写
2. **用户确认**：关键决策（阶段转换、项目结束）需要用户明确确认

```
所有必填项已填写 → AI 提示用户确认阶段转换
部分填写 → AI 提醒待完成的项
超时未完成 → AI 加急提醒
```

## 🔄 更改说明

本次从"静态文件系统"到"OpenClaw AgentSkill"的改造，是在**飞书对话框**中通过与 OpenClaw 助手（阿亮）的对话完成的。

整个过程没有使用传统的 IDE 或代码编辑器，而是：
- 在飞书中描述需求和约束
- 由阿亮执行具体操作（创建文件、调用 API、配置 cron、推送代码）
- 通过对话迭代调整方案（评估机制、驱动逻辑、文件结构等）
- 最终由阿亮完成 GitHub 推送

这本身就是一个 PDCA 循环的实践：Plan（对话规划）→ Do（对话执行）→ Check（对话验证）→ Act（确认和归档）。

## 💡 最佳实践

### 推荐部署方式

目前最佳实践是**在飞书中搭建 Channel，通过 OpenClaw 实现人机协作**：

1. **飞书作为项目管理界面**：用户在飞书知识库中填写 PDCA 文档，直观方便
2. **OpenClaw 作为驱动引擎**：通过 Cron 和 Heartbeat 主动推进流程
3. **飞书对话作为交互通道**：提醒、确认、决策都在飞书对话中完成

### 为什么选择飞书 + OpenClaw

| 能力 | 纯 Markdown 文件 | 飞书 + OpenClaw |
|------|-----------------|-----------------|
| 主动提醒 | ❌ 无法触发 | ✅ Cron + Heartbeat |
| 文档协作 | ❌ 本地文件 | ✅ 飞书知识库多人协作 |
| 进度判断 | ❌ 需要用户告知 | ✅ AI 主动读取文档 |
| 项目管理 | ❌ 手动维护 | ✅ 自动注册和清理任务 |
| 用户体验 | 需要打开编辑器 | 在飞书中直接操作 |

### 使用建议

1. **问题评估**：描述问题时先和 AI 聊背景，让它判断是否适合 PDCA
2. **文档填写**：AI 会创建模板并给出链接，按要求填写即可
3. **关注提醒**：每天早上会收到项目状态提醒，及时处理
4. **阶段转换**：当 AI 提示转换条件满足时，确认后自动进入下一阶段
5. **经验积累**：项目完成后自动归档到经验库
