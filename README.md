# PDCA 项目管理系统

**基于 PDCA 循环的 AI 增强问题解决系统** — OpenClaw AgentSkill，集成飞书全工具链，AI 主动驱动。

---

## 🎯 系统简介

PDCA 项目管理系统是一个**时间流驱动**的问题解决工作流系统。它将经典 PDCA 循环与 AI Agent 能力结合，通过飞书全工具链（Wiki + Bitable + Task + Calendar + Sheet）实现文档存储、进度可视化、任务管理、时间管理和数据分析的完整闭环。

### 核心特点

- **严格时间序列**：Plan→Do→Check→Act 按时间顺序推进，不可跳跃
- **AI 主动驱动**：Cron 定时提醒 + Heartbeat 心跳检查，项目不再停滞
- **飞书全工具链闭环**：Wiki 存文档、Bitable 存状态、Calendar 存时间、Task 存待办、Sheet 存数据
- **进展判断**：AI 主动读取飞书文档检查完成度，关键决策需用户确认
- **知识沉淀**：项目结束后经验自动归档到经验库，新项目可复用
- **制造质量专精**：内置设备 OEE、不良率、效率等场景模板

## 🔄 工作流程

```
用户描述问题
    ↓
AI 评估是否适合 PDCA（先聊背景，推荐一次不硬推）
    ↓
用户确认
    ↓
AI 自动创建：
  ├── Wiki 知识库（项目结构 + 阶段模板文档）
  ├── Bitable 看板（项目状态记录）
  ├── Calendar 日历（阶段截止日 + 里程碑）
  ├── Task 任务（执行计划待办）
  ├── Sheet 数据表（量化指标收集模板）
  └── Cron 定时任务（每日提醒 + 阶段预警）
    ↓
用户在飞书中工作（填写文档、录入数据、完成待办）
    ↓
AI 主动巡检（读取文档判断完成度、更新看板状态、检查待办进度）
    ↓
AI 推送提醒 → 用户确认决策（阶段转换 / 项目结束）
    ↓
循环直到完成 → 经验沉淀到 Wiki 经验库 → 清理 Cron
```

## ⚙️ 主动驱动机制

| 驱动方式 | 触发条件 | 行为 |
|---------|---------|------|
| Cron 每日提醒 | 每天 09:00 | 读取项目状态，提醒当前阶段待办、剩余时间 |
| Cron 阶段预警 | 阶段结束前1天 | 提示检查阶段转换条件 |
| Cron 里程碑提醒 | 里程碑当天 | 提醒待交付内容 |
| Heartbeat | 每次心跳轮询 | 轻量检查：有无活跃/到期/超时项目 |

**决策权始终在用户手中。** AI 只负责推动和信息，阶段转换、项目结束必须用户确认。

## 💡 使用建议

1. **问题评估**：描述问题时先和 AI 聊背景，让它判断是否适合 PDCA
2. **文档填写**：AI 会创建模板并给出飞书链接，按要求填写即可
3. **关注提醒**：每天早上会收到项目状态提醒，及时处理
4. **阶段转换**：AI 提示转换条件满足时，确认后自动进入下一阶段
5. **经验积累**：项目完成后自动归档到经验库，新项目可复用
