## 1. 概述

### 1.1 设计目标

基于飞书全工具链（Wiki、Bitable、Task、Calendar、Sheet），构建一个以 Bitable 为核心存储的 PDCA 任务/日程管理系统，实现：

1. **自动任务生成**：从问题分析阶段自动拆解任务，而非仅依赖手动执行计划
2. **表单数据收集**：支持便捷的多维数据录入
3. **实时仪表板**：单项目综合视图，实时更新
4. **智能巡检频率**：根据项目类型动态调整检查频率
5. **数据流闭环**：用户操作 → 数据存储 → AI 分析 → 洞察推送

### 1.2 核心原则

- **Bitable 为单一事实来源**：所有项目状态数据以 Bitable 为准
- **框架一致性**：任务拆分、数据收集、仪表板展示均延续 Plan 阶段选择的 MECE 框架
- **频率智能化**：巡检频率根据项目类型、阶段、紧迫度动态调整
- **用户友好**：通过表单、卡片等方式降低用户操作成本

## 2. 架构设计

### 2.1 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Bitable（核心存储）                       │
├─────────────────────────────────────────────────────────────┤
│  表1：项目主表                    │
│  表2：任务表（Tasks）             │
│  表3：数据收集记录表（Research Data）                        │
│  表4：执行日志表（Execution Logs）                           │
└─────────────────────────────────────────────────────────────┘
                            ↑ ↓ 自动同步
┌─────────────────────────────────────────────────────────────┐
│              飞书交互层（用户操作界面）                      │
├─────────────────────────────────────────────────────────────┤
│  • 飞书任务              │
│  • 飞书日历              │
│  • 飞书表单（Bitable Form）- 数据收集                       │
│  • 飞书卡片 - 交互式提醒                                    │
└─────────────────────────────────────────────────────────────┘
                            ↑ ↓ 实时更新
┌─────────────────────────────────────────────────────────────┐
│              AI 控制层                         │
├─────────────────────────────────────────────────────────────┤
│  • 问题分析 → 任务自动生成                                   │
│  • 框架维度任务拆分（MECE维度）                              │
│  • 进度巡检 → 状态同步                                       │
│  • 数据分析 → 洞察推送                                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 数据流

```
用户描述问题
    ↓
AI 评估 → 用户确认
    ↓
自动创建：
  Wiki（项目文档+模板）
  + Bitable（4张核心表）
  + Calendar（关键时间节点）
  + Task（具体待办）
  + Cron（定时巡检）
    ↓
用户在飞书中工作
  ├── 填写表单 → 数据记录表
  ├── 完成Task → 任务表
  └── 查看仪表板
    ↓
AI 定时巡检（智能频率）
  ├── 读取表单数据
  ├── 检查任务状态
  └── 分析趋势
    ↓
更新仪表板 + 推送洞察卡片
    ↓
项目结束 → 经验沉淀
```

## 4. 任务自动生成机制

### 4.1 生成流程

```
问题分析完成
    ↓
AI 提取问题要素（问题症状、影响范围、历史情况）
    ↓
根据选择的 MECE 框架，按维度拆解
    ↓
每个维度生成 1-N 个具体任务
    ↓
写入 Bitable 任务表
    ↓
同步创建飞书 Task
```

### 4.2 维度任务映射示例

#### TREND 框架（健康/体能）

| 问题维度 | 典型任务 |
|----------|---------|
| Training（训练） | 制定训练计划、购买训练设备、安排训练时间 |
| Rest（休息） | 建立睡眠追踪、规划休息日、设置睡眠提醒 |
| Eating（营养） | 制定营养方案、计算热量需求、采购营养补剂 |
| Nature（身体） | 安排体能测试、建立基线数据、预约体检 |
| Daily（日常） | 设置活动提醒、记录日常步数、规划作息 |

#### 4M1E 框架（制造业）

| 问题维度 | 典型任务 |
|----------|---------|
| Man（人员） | 安排培训、制定操作规范、考核技能 |
| Machine（设备） | 设备检修、备件采购、维护计划 |
| Material（材料） | 来料检验、库存盘点、供应商评估 |
| Method（方法） | SOP 制定、流程优化、作业指导书更新 |
| Environment（环境） | 温湿度控制、5S 整理、安全检查 |

### 4.3 任务生成规则

1. **数量规则**：每个维度生成 1-3 个核心任务
2. **优先级规则**：根据问题紧急程度分配任务优先级
3. **依赖规则**：识别任务间的前后依赖关系
4. **时间规则**：根据项目周期自动分配截止日期

## 5. 智能巡检频率策略

### 5.1 按项目类型的频率基线

| 项目类型 | 默认巡检频率 | 说明 |
|---------|-------------|------|
| **健康/体能（TREND）** | 每日 1 次 | 运动数据每天更新即可 |
| **制造业（4M1E）** | 每班次 1 次（8小时） | 跟随生产节奏 |
| **软件开发（PPTD）** | 每日 1 次 | 日报制度 |
| **学习/教育（COMET）** | 每周 1 次 | 周报制度 |
| **财务投资（3RL-TD）** | 每日 1 次 | 市场交易日 |
| **销售营销（5P）** | 每周 1 次 | 周报制度 |
| **团队协作（GRCT）** | 每周 1 次 | 周会节奏 |

### 5.2 动态频率调整因子

| 因子 | 触发条件 | 频率调整 |
|------|---------|---------|
| **阶段因子** | Do 阶段 | ×1.5（执行期需更密切跟踪） |
| **紧迫度因子** | 优先级=高 | ×2 |
| **时间因子** | 距截止 < 20% 时间 | ×2 |
| **进度因子** | 状态=预警/滞后 | ×2 |
| **用户覆盖** | 用户自定义设置 | 以用户设置为准 |

### 5.3 Cron 表达式示例

```bash
# 健身项目 - 每日早上 8 点检查
0 8 * * * pdca ongoing --project "个人VO2max提升"

# 制造项目 - 每 8 小时（每班次）
0 */8 * * * pdca ongoing --project "挖掘机臂架开裂改善"

# 紧急项目 - 每 4 小时
0 */4 * * * pdca ongoing --project "紧急设备修复"

# 学习项目 - 每周一早上 9 点
0 9 * * 1 pdca ongoing --project "Python学习计划"
```

### 6.2 仪表板实现方式

采用 **Bitable 视图 + Wiki 仪表板页面** 组合方式：

1. **Bitable 视图**：用于数据展示（表格、看板、甘特图等原生视图）
2. **Wiki 仪表板页面**：用于综合展示（AI 定期生成并更新）

### 6.3 点击下钻设计

| 入口 | 下钻目标 | 内容 |

### 7.2 表单创建流程

```
项目创建时
    ↓
AI 根据选择的框架 → 生成表单字段配置
    ↓
调用 Bitable API 创建表单视图
    ↓
生成表单分享链接 → 发送给用户
    ↓
用户填写表单 → 数据写入数据收集记录表
    ↓
Cron 检测到新数据 → 更新仪表板 + 触发分析
```
