## 1. 背景与问题

### 1.1 问题

此前采用 Bitable 作为项目统一管理方案，遇到以下问题：
- Bitable API 不支持在应用内自动创建/管理文档和文件夹
- Bitable 收集表、仪表盘、工作流的创建需要手动 UI 配置
- 数据导入存在不稳定问题

### 1.2 目标

- **统一数据源**：一个项目 = Wiki 下一个文件夹，所有文档和数据在一个地方管理
- **完全自动化**：AI 通过飞书 API 自动创建文档、读取内容、生成统计、推送消息
- **手动零配置**：全程由 AI 完成，用户不需要在飞书 UI 中手动配置任何东西

### 1.3 核心载体

| 功能 | 载体 | API |
|------|------|-----|
| 项目文档 | Wiki 文档（文件夹结构） | `feishu_create_doc`, `feishu_fetch_doc`, `feishu_update_doc` |
| 项目看板（总览） | Feishu Sheet | `feishu_sheet.create`, `feishu_sheet.read`, `feishu_sheet.append`, `feishu_sheet.update` |
| 数据收集记录 | Feishu Sheet | 同上 |
| 任务追踪 | Feishu Task | `feishu_task_task.create`, `feishu_task_task.list`, `feishu_task_task.patch` |
| 问卷调查 | 飞书表单 | AI 发送表单链接，数据写入 Sheet |

## 2. 系统架构

### 2.1 Wiki 文件组织结构

```
飞书 Wiki: PDCA项目管理（根节点）
├── 📊 项目看板（Sheet — 所有项目一览）
├── 📚 经验库/
│   ├── 经验索引（文档）
│   └── 各分类经验文档
│
└── 📁 [项目名称]/（每个项目一个 Wiki 子节点）
    ├── 📋 项目信息.md
    ├── 📁 Plan/
    │   ├── 问题分析.md
    │   ├── 目标设定.md
    │   ├── 解决方案.md
    │   └── 执行计划.md
    ├── 📁 Do/
    │   ├── 执行日志.md
    │   ├── 进度报告.md
    │   └── 问题记录.md
    ├── 📁 Check/
    │   ├── 数据分析.md
    │   ├── 效果评估.md
    │   └── 问题诊断.md
    ├── 📁 Act/
    │   ├── 行动决策.md
    │   ├── 实施方案.md
    │   └── 知识总结.md
    └── 📊 数据收集记录（Sheet）
```

### 2.2 数据流

```
用户描述问题
    ↓
AI 评估 → 用户确认
    ↓
自动创建：
  Wiki 项目文件夹（含阶段文档）
  + 项目看板 Sheet 新增一行
  + 数据收集 Sheet
  + 飞书任务（初始任务列表）
  + 飞书日历（里程碑）
  + Cron（定时巡检）
    ↓
用户填写文档和问卷
    ↓
AI 定时巡检（Cron agentTurn）
  ├── feishu_fetch_doc → 分析文档完成度
  ├── feishu_sheet.read → 读取数据收集记录
  ├── feishu_task_task.list → 检查任务完成率
  ├── 更新项目看板 Sheet
  └── 发送飞书消息卡片
    ↓
用户确认决策 → AI 更新文档和看板
    ↓
项目结束 → 经验沉淀到 Wiki
```

## 3. 数据结构设计

### 3.1 项目看板 Sheet

**位置**：Wiki 根节点下的一个独立 Sheet

**列结构**：

| 列名 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 项目ID | 文本 | 唯一标识 | PRJ-001 |
| 项目名称 | 文本 | 项目标题 | 个人VO2max提升 |
| 选择框架 | 文本 | MECE框架 | TREND |
| 当前阶段 | 文本 | PDCA阶段 | Plan |
| 状态 | 文本 | 正常/预警/超时/已完成 | 正常 |
| 负责人 | 文本 | 负责人姓名 | 张三 |
| 开始日期 | 日期 | 项目启动 | 2026-04-01 |
| 预计结束日期 | 日期 | 计划结束 | 2026-05-01 |
| 完成度 | 数字 | 0-100 | 65 |
| 剩余天数 | 数字 | 距截止日剩余 | 12 |
| 项目链接 | 文本 | Wiki 项目节点 URL | https://xxx |

### 3.2 数据收集 Sheet

**位置**：每个项目文件夹下创建

**结构**：

| 列名 | 类型 | 说明 |
|------|------|------|
| 记录ID | 文本 | 唯一标识 |
| 指标名称 | 文本 | 数据项名称 |
| 维度 | 文本 | 框架维度 |
| 数值 | 数字 | 测量值 |
| 单位 | 文本 | 测量单位 |
| 记录时间 | 日期时间 | 记录时间 |
| 记录人 | 文本 | 记录人员 |
| 数据来源 | 文本 | 手动录入/问卷/自动 |

创建时按 MECE 框架的维度动态添加对应的维度值到筛选或备注。

## 4. 自动化巡检流程

### 4.1 Cron agentTurn 每日巡检（09:00）

```
1. 读取项目看板 Sheet → 获取所有"进行中"项目
2. 对每个项目：
   a. 读取 Wiki 项目节点 → 获取各阶段文档
   b. feishu_fetch_doc 逐个读取文档内容
   c. 分析完成度（检查是否有实质性内容）
   d. 读取数据收集 Sheet → 分析数据趋势
   e. feishu_task_task.list → 检查任务完成率
   f. 计算时间进度偏差
   g. 更新项目看板 Sheet（完成度、状态、剩余天数）
3. 生成巡检消息：
   - 正常 → 简短文本概要
   - 预警/超时 → 飞书消息卡片（含链接和原因）
```

### 4.2 阶段到期预警（systemEvent）

阶段截止日前 1 天触发：检查阶段文档完成度，提醒用户确认转换条件。

## 5. 项目创建完整流程

```
PlanAgent Phase 4（自动执行）：

1. Wiki 创建
   - feishu_create_doc: 项目文件夹（在 PDCA项目管理 根节点下创建子节点）
   - feishu_create_doc: 项目信息.md + 各阶段文档（带模板内容）
   - 使用 wiki_node 参数确保文档在正确位置

2. 项目看板初始化
   - feishu_sheet.append: 在项目看板 Sheet 新增一行

3. 数据收集 Sheet 创建
   - feishu_sheet.create: "数据收集记录"（在项目文件夹下）

4. 任务创建
   - feishu_task_task.create: 按 MECE 框架维度生成初始任务

5. 日历事件
   - feishu_calendar_event.create: 里程碑和阶段截止日

6. Cron 注册
   - 创建每日巡检 agentTurn
   - 创建阶段到期预警 systemEvent
```

## 6. 与现有方案对比

| 方面 | Bitable 方案 | Wiki + Sheet 方案 |
|------|-------------|------------------|
| 项目文档 | Bitable 内文档视图（无 API） | Wiki 文档（完全 API 支持） |
| 项目看板 | Bitable 看板（仪表盘组件） | Sheet（API 完全支持） |
| 数据收集 | Bitable 收集表 | Sheet + 飞书问卷 |
| 任务管理 | Bitable 任务表 | Feishu Task（API 支持） |
| 自动化巡检 | Bitable 工作流 | Cron + AI 读取文档 |
| API 支持度 | 部分（文档无 API） | 全部支持 |
| 手动配置 | 需要（表单/仪表盘/工作流） | 零（全部自动化） |

## 8. 风险与限制

| 风险 | 缓解措施 |
|------|---------|
| Wiki 文档内容较长，AI 读取慢 | 按阶段分文档，不合并为单个长文档 |
| Sheet API 不支持公式字段 | 项目看板的"剩余天数""完成度"由 AI 计算后写入单元格 |
| 问卷数据自动写入 Sheet | AI 读取问卷提交数据，追加到数据收集 Sheet |
| Wiki 文件创建有频率限制 | 创建时串行调用 + 延迟 0.5-1 秒 |
