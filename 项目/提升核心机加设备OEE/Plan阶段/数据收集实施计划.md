# OEE数据收集实施计划

## 一、收集目标
通过为期8天的数据收集，量化分析当前OEE损失的根本原因，为制定改进措施提供数据支撑。

## 二、收集时间安排
- **收集周期**：2025-12-10 至 2025-12-17（共8天）
- **覆盖班次**：白班、夜班全覆盖
- **设备范围**：DMG-01、DMG-02、DMG-03、DMG-04、DMG-05五台设备

## 三、人员分工

### 数据收集小组
| 职责 | 负责人 | 备注 |
|-----|--------|------|
| 总协调 | 张主管 | 统筹数据收集工作 |
| 换型数据记录 | 李操作员、孙操作员 | 记录每次换型时间（每人负责2-3台） |
| 缺料数据记录 | 王物料员 | 记录物料供应情况 |
| 故障数据记录 | 赵维修员、钱维修员 | 记录设备故障情况（每人负责2-3台） |
| 生产数据记录 | 班组长 | 每日产量、质量数据 |
| 数据汇总分析 | 工程师 | 每日数据整理分析 |

### 数据收集培训
- **培训时间**：2025-12-10 上午9:00-10:00
- **培训地点**：生产现场会议室
- **培训内容**：
  1. OEE概念和数据收集意义
  2. 各表格填写规范
  3. 异常情况处理流程
  4. 数据质量要求

## 四、数据收集流程

### 1. 每日流程
**班前会（8:00）**
- 确认当日生产计划
- 分发数据记录表格
- 明确数据收集重点

**生产过程中**
- 实时记录换型时间
- 及时登记缺料情况
- 立即记录设备故障
- 定期巡查数据填写

**班后会（17:00）**
- 汇总当日数据
- 检查数据完整性
- 提交数据表格
- 简要说明异常

### 2. 数据验证机制
- **一级验证**：班组长每日检查
- **二级验证**：主管每周审核
- **三级验证**：工程师交叉验证

## 五、数据质量要求

### 准确性要求
- 时间记录精确到分钟
- 数量统计精确到个位
- 原因描述具体明确

### 完整性要求
- 所有停机必须记录
- 所有换型必须记录
- 所有故障必须记录

### 及时性要求
- 事件发生后30分钟内记录
- 每日数据当日汇总
- 异常情况立即报告

## 六、支持工具

### 1. 物理工具
- 马表（用于精确计时）
- 记录板（现场快速记录）
- 文件夹（表格存放）

### 2. 数字工具
- Excel表格（电子化存储）
- 共享文件夹（实时更新）
- 分析软件（初步分析）

## 七、可能遇到的问题及对策

### 问题1：人员配合度不高
**对策**：
- 加强培训，明确意义
- 建立激励机制
- 主管亲自督促

### 问题2：数据记录不及时
**对策**：
- 设置提醒机制
- 简化记录流程
- 定时检查提醒

### 问题3：数据准确性存疑
**对策**：
- 多重验证机制
- 现场抽查核对
- 视频监控辅助

### 问题4：生产任务繁忙，无暇记录
**对策**：
- 专人协助记录
- 优化记录方式
- 利用空闲时间填写

## 八、数据整理计划

### 每日整理
- 20:00前完成当日数据录入
- 计算当日OEE
- 识别异常数据

### 每周整理
- 周末汇总周数据
- 分析OEE变化趋势
- 编制周报

## 九、预期成果

1. **获得8天完整的OEE数据**，包括：
   - 详细的停机时间分类统计
   - 换型时间分布情况
   - 缺料影响量化数据
   - 故障频率和维修时间

2. **识别OEE损失的主要因素**，确定改进重点

3. **建立 baseline 数据**，用于后续改进效果对比

4. **形成标准化数据收集流程**，为持续改进奠定基础

## 十、下一步行动
1. 召开数据收集启动会议
2. 分发数据收集表格
3. 开始为期8天的数据收集
4. 每日跟踪数据收集进度
5. 收集完成后进行数据分析
